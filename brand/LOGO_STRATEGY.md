# 人格麻將 Logo 策略与方案

> 本文档记录 logo 的完整思考过程、约束、方向分析、我的方案与推荐。
> 配套 SVG 概念稿见 `brand/concepts/`，可视化预览见 `brand/preview.html`。
> 日期：2026-07-17

---

## 1. 需求梳理（老板 brief + 用户补充）

**老板明确要求（WhatsApp）：**
1. **无文字**：一个中英（及未来其他语言）通用的单一 logo，不要任何汉字/字母。
2. **概念要灵活**：未来会有多种人格模型（不止 5 维，可能 4 维、6 维），设计**不能锁死在固定的形状数量/角数**（例如不要五边形）。
3. **必须含 Ψ (Psi)**：心理学 / 心智的通用希腊字母与科学符号。老板对这一点态度很坚定。

**老板后来 AI 生成、且明确表示喜欢的两个方向：**
- **方向 A**：卡牌 + 脑子 + Ψ（脑神经网络里长出一个 Ψ，带节点/连线，科技感）。
- **方向 B**：Ψ + **鲁宾花瓶错觉（Vase/Two Profiles）** + 扁平卡通化风格。

**用户（你）的判断：**
- 不要太**写实**——我们网站是偏卡通/暖色手绘风，写实 logo 会突兀、也太难懂。
- 感觉老板一定想要 **Ψ 符号**。
- 让我基于这两个他喜欢的方向 + **贴合我们网页配色**来做。

---

## 2. 核心洞察（为什么这几个元素能拼到一起）

| 元素 | 含义 | 满足哪条要求 |
|---|---|---|
| **Ψ (Psi)** | 心理学 / 心智 | 老板硬需求；语言无关 |
| **麻将牌身**（圆角矩形 + 象牙金边） | 游戏本体；呼应我们卡框视觉 | 贴合网站；点明「麻将」 |
| **鲁宾花瓶 / 双侧脸** | 「先读懂自己，再读懂别人」+ 知觉的双重性 | 正好是游戏核心循环的隐喻 |
| **圆形轨道 / 自由数量的点** | 多种人格模型（4/5/6 维皆可） | 老板「不锁死维度数」硬需求 |
| **暖琥珀金 + 象牙 + 陶土红** | 和网站 `--psy-*` 一致 | 用户要求；避免冷银蓝 |

> **关键发现 1：Ψ 本身就长得像一个高脚杯 / 花瓶。** 所以「Ψ + 鲁宾花瓶」几乎是天作之合——
> 只要在 Ψ 两侧补上两张对望的侧脸，中间的 Ψ 立刻被读成花瓶。老板的方向 B 是最聪明的一条线。
>
> **关键发现 2：用「圆形」而不是「多边形」承载维度，就天然绕开了「锁死维度数」的问题**——
> 无论 4/5/6 维，圆环上放几个点都不突兀。这正是老板自己那段长描述里反复强调的。

---

## 3. 我的设计原则（把两条方向落进我们的风格）

1. **暖不冷**：全部改用我们网站的琥珀金 `#c39a52` / 象牙 `#fdf9f0` / 陶土 `#c9603f`，**放弃**老板 AI 稿里的冷蓝银。
2. **扁平、少即是多**：粗描边、圆角、纯色块，能在 favicon（16–32px）尺寸下仍认得出 Ψ。写实和复杂神经网络在小尺寸会糊，弱化。
3. **双层品牌系统**（重要建议）：
   - **主标记（icon / 语言无关 logo）**：越简单越好 → 用「Ψ + 麻将牌」。做 favicon、App 图标、页首。
   - **徽记（emblem，讲故事）**：信息更丰富 → 用「Ψ + 花瓶双脸」或「Ψ + 圆形轨道」。做加载页、结算分享图、规则大图。
   - 一简一繁，各司其职，避免「既要小尺寸清晰又要信息丰富」的自相矛盾。

## 4. 四个方案（详见 `concepts/` 与 `preview.html`）

### 方案 A · Psi Tile（主标记，**推荐做默认 logo / favicon**）
- **构成**：一张象牙金边的麻将牌，正中一个金色 Ψ。
- **优点**：最简洁、最耐缩放；直接点题「麻将 × 心理」；不含任何维度数量元素，完全灵活；和网站卡框同一套视觉语言，放上去像「本来就长在网页里」。
- **用途**：favicon、App/PWA 图标、页首标记、PVP 房间角标。

### 方案 B · Psi Vase（徽记，**推荐做加载页 / 分享图**）
- **构成**：暖色圆徽，两张陶土色对望侧脸，中间的负空间是一只金色花瓶——而花瓶就是 Ψ。
- **优点**：老板方向 B 的暖色化；叙事最强（读己读人 + 知觉双重性）；语言无关；圆形无维度锁定。
- **注意**：细节多，小尺寸会糊，**只用在中大尺寸**。

### 方案 C · Psi Orbit（徽记，灵活维度的最佳表达）
- **构成**：中心 Ψ，外围一圈「圆形轨道」，轨道上散布几枚小麻将牌 + 小圆点（数量只是示意）。
- **优点**：把老板自己那段「圆形星系、未来加多少维度都合适」的描述直接可视化；动态感强。
- **注意**：可做静态，也非常适合做加载动画（轨道旋转）。

### 方案 D · Psi Mind（呼应老板方向 A，已简化）
- **构成**：圆徽内金色 Ψ，顶部一道虚线脑半球弧 + 几个节点圆点（神经/数据连接），暖色。
- **优点**：保留老板喜欢的「脑 + Ψ + 节点」科技感，但砍掉复杂神经网络，改暖色、可缩放。
- **注意**：节点数量随意，不锁维度。

---

## 5. 推荐

- **主 logo（默认、favicon、页首）：方案 A（Psi Tile）。** 简单、通用、耐缩放、最贴网站。
- **徽记（加载页、结算分享、规则大图）：方案 B（Psi Vase）或方案 C（Psi Orbit）二选一。**
  - 想要「故事感 / 高级感」→ B。
  - 想要「灵活维度 + 动效潜力」→ C。
- 方案 D 作为给老板的「你喜欢的脑方向我也做了暖色简化版」备选，好交代。

> 一句话给老板：**「Ψ 一定保留；主 logo 用最简的『Ψ + 麻将牌』保证到处都清晰，另配一个『Ψ 花瓶双脸』徽记讲故事。全部换成网页的暖金色，不锁维度数。」**

## 6. 配色 token（和 `globals.css` 对齐）

| 用途 | 变量 | Hex |
|---|---|---|
| 金（Ψ 主色） | `--psy-accent` | `#c39a52` |
| 深金（描边/阴影） | `--psy-accent-strong` | `#9a7448` |
| 象牙（牌面） | `--psy-surface` / card-content | `#fdf9f0` / `#fdf8f1` |
| 暖沙（背景/融入） | `--psy-bg` | `#f4edd9` |
| 墨（深色描边可选） | `--psy-ink` | `#3a3020` |
| 陶土红（侧脸/点缀） | `--psy-danger` | `#c9603f` |
| 柔绿（点缀，慎用） | `--psy-success` | `#6f8f55` |

维度色（O `#c084fc` C `#60a5fa` E `#facc15` A `#4ade80` N `#f87171`）**不建议进 logo**——会把 logo 和「5 维」绑死，违反「灵活维度」要求。留给游戏内 UI 即可。

---

## 7. Do / Don't

**Do**
- Ψ 永远是主角、居中、可辨。
- 圆形 / 牌形承载，避免多边形。
- 暖色，和网页同源。
- 先保证 16px favicon 认得出。

**Don't**
- 不要写实（手、真实大脑、3D 金属反光）。
- 不要五边形 / 固定 N 个角。
- 不要冷蓝银配色。
- 不要在标记里放文字（页首可用「标记 + 文字」锁版，但**文字不属于 logo 本体**）。
