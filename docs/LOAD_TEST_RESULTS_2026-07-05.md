## 结论与决策（对应 PLAN §3 决策表）

1. **单堂 200 人课，free tier 技术上扛得住**：连接、写入、广播延迟、断线恢复全部远优于阈值。
   游戏内延迟 p95 142ms，玩家无感；host-as-master 架构的 resync 兜底在压力下真实有效。
2. **仍建议上课当月升 Pro（$25/mo），理由变了**：
   - 不是延迟/并发崩，而是 **(a)** 200 并发是软配额（220 实测放行），fair-use 政策下不可依赖，
     正式课贴线跑没有 SLA 保障；**(b)** **消息额度才是先触顶的墙**：实测外推一堂课 ≈ 24.5 万条，
     free 2M/月 ≈ **8 堂课**，一学期多班/多周必超。
3. **Vercel 不用升**（对局流量不经 Vercel），维持原判。
4. **冒烟阶段看到的 1-10s 延迟尖峰确认为冷启动现象**，正式规模稳定复现不出来。
   → 开课日 runbook：提前 10 分钟访问一次唤醒项目 + 开一局预热。
5. **剩余唯一未验证风险 = 教室 WiFi（T5）**：200 台设备挤同一 AP 的无线层，只能真机彩排
   （见 PLAN §1-T5 流程）。云端这头已经清了。
