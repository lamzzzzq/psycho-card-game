# 卡牌美术规格 · 交给 AI 生图用

> 2026-06-21 定稿。真相源：`src/components/game/TarotCard.tsx`（卡面渲染）、`src/data/questions.ts`（50 题↔人格牌）、`src/data/dimensions.ts`（维度配色）、`src/data/dummy-cards.ts`（知识牌）。
> 改了卡面比例/图窗占比，记得同步更新本文与生图比例。
>
> 牌分两类：**人格牌**（50 张，需插画，§1–§5）、**知识牌/dummy**（20 张，纯文字、无需插画，§6）。**生图只针对 50 张人格牌。**

---

## 1. 要生成什么

**50 张人格牌插画**，对应 IPIP-50 的 50 道题（id 1–50，5 维度各 10 题）。每张图 = 该题描述的人格情景。

| 项 | 规则 |
|----|------|
| 数量 | 50 张（id 1–50） |
| 命名 | `{id}.webp` —— `1.webp`、`2.webp` … `50.webp` |
| 放置 | `public/cards/`（**目录需新建**，放进去即自动替换占位 ◈） |
| 映射 | id↔题面见 `src/data/questions.ts`。例：`1.webp`→「我是派對中的靈魂人物」(外向)，`5.webp`→「我詞彙豐富」(开放) |

> 4 人局会用到 80 张牌（>50 时按维度复制题面，副本是新 id、无对应图 → 回退占位，不影响玩法）。先做 50 张主图即可。

---

## 2. 卡面结构（决定构图）

卡面是 **CSS 渲染的塔罗框，插画只填中间「图窗」**，不是整张卡：

```
┌───────────────────┐  ← 金色双线框 + 顶角 ✦ 星点（CSS，不在图里）
│   ┌───────────┐   │
│   │   插画区   │   │  ← 图窗：你的插画放这里（占卡片高度 60%）
│   │           │   │     object-cover 居中裁切填满
│   └───────────┘   │
│    ──── ◆ ────    │  ← 分隔线 + 菱形（CSS）
│      题目文字      │  ← 题面文字（CSS 叠加，占剩余 40%，不在图里）
└───────────────────┘
```

**插画里只画主体本身**：不要画卡框/边框/星点 ✦/菱形 ◆、不要写任何文字——这些全是代码叠加层，画进去会和 CSS 框穿帮。主体别贴边（金框会压住四边）。

---

## 3. 比例（关键）

- **整张卡 = 4:7**（高/宽 ≈ 1.75）。手牌与「看大图」场景（抽到的牌 / 弃牌堆放大 / card-lab）**统一同一比例**，不再有细长 1:2。
- **图窗 = 卡内宽约 92% × 卡高的 60%** → 实际图窗比例 **≈ 0.92:1（近正方、略高，高/宽≈1.09）**。
- 渲染 = `object-cover`（居中裁切填满）。

**→ 生图出 1:1 正方形（1024×1024）。** 主体居中、四周留 ~10% 安全边——窗口比正方略高，`object-cover` 会裁掉**左右各约 4%**。

> 历史：曾短暂用过 1:2 卡（图窗≈2:3，建议出 1024×1536），现已废弃。**别再用 1024×1536**，会被近正方窗口裁掉上下。

## 4. 插画风格与主角分配

实际采用 **3D Soft Clay Cartoon Archive**：半 3D 软黏土/纸艺微缩场景、哑光触感、圆润但保持成年比例的人物、黑色点状或短线眼睛、暖米色住宅、木家具与植物、柔和自然光、低至中等对比。

- **25 女 / 25 男**：奇数 id 使用同一位女性主角，偶数 id 使用同一位男性主角；五个维度各 5 女 5 男。
- **女性主角**：深色及肩微卷发、陶土红帽衫、深灰长裤。
- **男性主角**：黑色短卷发、深绿色帽衫、深色长裤。
- 两位主角都必须是年轻成年人；不要幼态比例、夸张大头、动漫/Disney 眼睛或性别刻板化服装。
- 情景由题意决定；性别切换时只替换主角身份和服装色，不改动作、构图、配角、道具或场景含义。
- 卡面的暗蓝、米黄、暗金、古籍与金属质感仍由 CSS 外框承担，不画进插画。

## 5. 落地清单

1. 生 50 张 **1024×1024**，主体居中留边，不画框/不写字。
2. **转 webp**（OpenAI 出 PNG，代码只认 `.webp`）：`cwebp in.png -o {id}.webp` 或用 sharp 批处理。
3. 命名 `1.webp`…`50.webp`，放进 `public/cards/`。
4. 刷新 `/card-lab` 或牌桌即生效（缺图自动回退 ◈ 占位）。

> 需要 50 张逐题的生图 prompt 清单（题面 + 维度 + 建议画面，CSV/MD），可让 AI 按本规格一次生成。

## 6. 知识牌（dummy）· 纯文字，无需插画

知识牌是手牌里的「噪音牌/中立牌」，不计入人格归档，同时承载 APSS 课程知识点。**不画插画**，走专属纯文字版式。

- **数量/内容**：20 张课程心理学概念，**术语(term) + 一句话定义(definition)**，统一英文。真相源 `src/data/dummy-cards.ts`（`KNOWLEDGE_CARDS`，来自课程 sheet A 列术语 + C 列 Bryant 定义）。
- **版式**（同 4:7 外框，但内部无图窗）：眉标 `Psyche` + **术语标题** + 分隔 ◆ + **定义正文**（小字）。渲染走 `TarotCard` 的 `description` 入参触发。
- **与人格牌区分**：人格牌有图窗 + 一句陈述；知识牌纯文字 + 术语标题，一眼可辨。
- **发牌**：每局抽 8（2–3 人）/ 12（4 人）张，从 20 条按序循环取。

> 现状统一英文（中文模式下也显示英文术语，课程通用）。若日后要中文对照，在 `KNOWLEDGE_CARDS` 加字段 + `TarotCard` 知识牌分支按 locale 取即可。
