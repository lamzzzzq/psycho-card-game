# 罚停机制 · Edge Case 手动测试清单

> 用途：6 月初测试前，逐条手动验证罚停（碰失败 / 自摸碰失败 / 食胡失败）的边界行为。
> 真相源代码：`src/lib/game-logic.ts`。自动化测试：`src/lib/__tests__/penalty-freeze.test.ts` + `pong-empty-hand.test.ts`（共 84 条，全绿）。
>
> 测试建议用 **3 人联机**（PVP 默认 3 人 5 轮），座位记为 A→B→C。

---

## 0. 先理解罚停的「三个标记」

一次失败会给犯规者打上最多三个内部标记：

| 标记 | 含义 | 何时清除 |
|------|------|----------|
| `skipNextTurn` | 下一个「自己的回合」被自动跳过 | 被自动跳过时清除 |
| `extraSkipQueued`（加重罚停） | 第一次跳过后**再激活一次** skipNextTurn | 第一次跳过时消费 |
| `frozenUntilOwnDiscard` | 从失败那刻起锁死**所有** claim-window（不能碰/胡/skip） | 自己真正完成一次 **draw+discard** 后清除 |

**净效果：一次失败 = 罚停约「一整轮 + 2 个自己的回合」**，期间不能参与任何抢牌，直到自己重新出牌一次才完全解禁。

暴露手牌（reveal）：
- **碰失败 / 自摸碰失败** → 只亮出押错的那几张（`revealedSelectedCards`）
- **食胡失败** → 亮出**整手牌**（`revealedHand`，惩罚更重，匹配宣胡的赌注）

reveal 在犯规者**下次真正摸牌**时清除（要熬过整个罚停期）。

## 1. 碰失败（别人弃牌时抢碰，pong-fail）

### CASE 1.1 — 张数不够也算失败（不是无效点击）
- **设置**：你的 target O = 3（BigFive O≈3）。A 弃出一张 O。
- **操作**：你只选 1 张手牌 O 就确认碰（总共 1+1=2 < 3）。
- **预期**：判**碰失败**，不是「没反应」。你被罚停；亮出你选的那 1 张；轮次继续。
- **反例（要确认不会发生）**：旧版本会「静默无反应」，现在必须真的扣罚。

### CASE 1.2 — 一张不选就确认
- **操作**：claim-window 里不选任何手牌直接确认碰。
- **预期**：碰失败 + 罚停；`revealedSelectedCards` 为空数组（亮出 0 张，但仍然罚停）。

### CASE 1.3 — 张数对但维度混错
- **设置**：target O=3。A 弃 O。你手里有 1 张 O + 1 张 C。
- **操作**：选这 2 张 + 桌上 O = 3 张确认。
- **预期**：张数对但有 C → 碰失败 + 罚停；亮出这 2 张（**不亮整手牌**）。

### CASE 1.4 — 强 trap：碰一个自己已经归档的维度
- **设置**：你已经归档过 O。A 又弃一张 O。
- **操作**：你仍然选牌确认碰 O。
- **预期**：碰失败 + 罚停，失败原因记为 `already-declared`（数据里可区分）。UI 上「已归档维度」的碰按钮应**视觉降级**但仍可点（点了就中 trap）；选卡前取消则无惩罚。

### CASE 1.5 — 只暴露押错的牌，不暴露整手
- **预期**：1.1~1.4 任意碰失败后，其他玩家只看到你押的那几张翻开，**手里其余牌仍盖着**。

## 2. 自摸碰失败（自己回合碰，self-pong-fail）

### CASE 2.1 — 自摸碰选错 → drawnCard 退回手牌
- **设置**：轮到你，已摸牌（drawnCard 在手）。target O=3，你选 2 张错维度的牌提交自摸碰。
- **预期**：
  - 自摸碰失败 + 罚停（skipNextTurn + frozenUntilOwnDiscard 都置位）。
  - **刚摸的那张牌退回手牌**（不让你靠「弃掉它」来立即解冻）。
  - 立刻**让出本回合**，轮到下家；`drawnCard` 清空。

### CASE 2.2 — 每回合只能自摸碰一次
- **操作**：本回合已经自摸碰成功过一次，再点自摸碰。
- **预期**：第二次**静默无效**（不算失败、不罚停）。这是硬规则，优先于 trap 判定——避免玩家被莫名其妙罚。
- **解禁**：下回合摸牌后名额恢复。

### CASE 2.3 — 强 trap：自摸碰已归档维度
- **设置**：已归档 O，本回合还没用过自摸碰名额，再提交自摸碰 O。
- **预期**：自摸碰失败 + 罚停，原因 `already-declared`。

## 3. 食胡失败（attemptHu fail）

### CASE 3.1 — 自己回合宣胡但牌不够（phase=drawing，还没摸牌）
- **操作**：轮到你、还没摸牌就点「胡」，但手牌凑不齐所有未归档维度。
- **预期**：
  - 食胡失败 → **立刻 advance turn**，下家接管（**不再停在你这**）。
  - 你被罚停（skipNextTurn + frozenUntilOwnDiscard）+ **整手牌翻开**。

### CASE 3.2 — 自己回合宣胡失败（phase=discarding，已摸牌）
- **预期**：同 3.1，且**刚摸的牌退回手牌** + advance turn + drawnCard 清空。

### CASE 3.3 — banner 与按钮一致性（旧 bug 回归）
- **要确认的反例**：旧版本食胡失败后 banner 显示「罚停」，但按钮没禁、当回合还能 draw+discard，且那次 discard 会**立即清掉** frozenUntilOwnDiscard。
- **现在预期**：失败后当回合直接结束，frozenUntilOwnDiscard 一定保留到下一圈自己的解冻回合，不会被当回合误清。

### CASE 3.4 — 在别人 claim-window 里宣胡失败
- **设置**：A 弃牌，claim-window 开着，你（非 A）点胡但牌不够。
- **预期**：记为你已响应（claimResponses 含你），等其他人响应完再 finalize；你被罚停 + 整手翻开。

> 注：食胡**成功**可以在别人 claim-window 里发生（手牌已满足全维度即可胡，不需要桌上那张），立即结束游戏判你赢。

## 4. 罚停的「冻结」传播（最容易出 bug 的区）

### CASE 4.1 — 受罚者不会卡死别人的 claim-window
- **设置**：A 弃牌 → C 碰失败（claim-window 结束）→ 轮到 B。
- **操作**：B 摸牌 + 弃一张人格牌 → 新 claim-window 打开。
- **预期**：C 仍在罚停 → **自动记为已 skip**，窗口不会卡住等 C；只等 A 响应。
  - ⚠️ 这是关键死锁点：受罚者的抢牌面板是隐藏的，没有自动 skip 就会永久卡住。

### CASE 4.2 — 受罚者在冻结期不能碰
- **设置**：C 处于 frozenUntilOwnDiscard/skipNextTurn 状态，B 弃出一张 C 本可以碰的牌。
- **预期**：C 点碰**静默无效**（被冻结挡住）。

### CASE 4.3 — 受罚者在冻结期不能胡
- **预期**：同 4.2，点胡也无效。

### CASE 4.4 — 完整一圈：A→B→C→A 后 C 解禁
- **设置**：A 弃牌 → C 碰失败。
- **操作顺序**：
  1. B（另一抢牌人）skip → 窗口结束，轮到 B。
  2. B 摸+弃人格牌 → claim-window，C 自动 skip。
  3. A skip → finalize，轮到 C → C 被自动跳过（消费一次 skipNextTurn）→ 落到 A。
- **预期**：因为**加重罚停**，C 第一次被跳后 `skipNextTurn` 又被重新激活（还要再跳一次）；`extraSkipQueued` 已消费为 false。C 仍未解禁。

### CASE 4.5 — frozenUntilOwnDiscard 熬过自动跳过仍然有效
- **预期**：犯规者被自动跳过（消费 skipNextTurn）之后，frozenUntilOwnDiscard **仍然 true**——只有自己真正出一次牌才解。验证：被跳过后的下一个 claim-window 里他仍被自动 skip。

### CASE 4.6 — 自己出一次牌才解冻（人格牌）
- **预期**：犯规者熬完跳过、轮到自己 draw+discard 一张**人格牌**后，frozenUntilOwnDiscard 变 false，恢复正常。

### CASE 4.7 — 弃 dummy 牌也能解冻
- **预期**：解冻条件是「自己完成一次 draw+discard」，弃 **dummy 牌**同样算数（dummy 不开 claim-window 但仍是一次出牌）→ 解冻。

### CASE 4.8 — reveal 熬到下次摸牌才清
- **预期**：押错翻开的牌 / 整手翻开，要熬过整个罚停期，直到犯规者**下次真正摸牌**那一刻才盖回去。

### CASE 4.9 — 多人同时受罚
- **设置**：A 弃牌，B 和 C 都处于罚停。
- **预期**：B、C 都被自动 skip，窗口在剩下的人那里 finalize，不卡死；turn 推进到合理位置不卡。

## 5. 加重罚停的精确轮数（self-pong-fail 最重）

| 失败类型 | claim-window 冻结 | 自己回合跳过 | 解禁动作 |
|----------|------------------|-------------|----------|
| 碰失败 | 全程冻结 | 2 个 own-turn（含 extraSkip） | 第 2 个 own-turn 后那次 draw+discard |
| 食胡失败 | 全程冻结 + 整手翻开 | 2 个 own-turn | 同上 |
| 自摸碰失败 | 全程冻结 | 2 个 own-turn | 最重（已先看到摸的牌 + 选了维度，信息最多） |

### CASE 5.1 — 验证「跳 2 个自己的回合」
- **操作**：制造一次碰失败，然后一路观察该玩家。
- **预期**：他**连续两个**轮到自己的回合都被自动跳过（不是一个），第二次跳过后 skipNextTurn 才彻底清。

## 6. 新修复：碰后空手死锁（CASE 6，本次新增）

### CASE 6.1 — 碰光最后两张牌
- **设置**：你手里只剩 2 张 O（target O=3），其他维度还没全归档。A 弃一张 O。
- **操作**：碰 O（2 张手牌 + 桌上 1 张 = 3 张，成功）。
- **预期**：归档 O 成功后**手牌清空**，但你没胡（还有维度没归档）。
  - ✅ 修复后：你的偷得的回合**直接结束**，轮到下家——不会卡在「该你弃牌但没牌可弃」。
  - ❌ 修复前：会停在 `discarding`、空手、无法操作 → 永久死锁。

### CASE 6.2 — 自摸碰碰光手牌+刚摸的牌
- **设置**：手里 2 张 O，刚摸的也是 O，target=3。
- **操作**：自摸碰 O 选这 3 张（成功）。
- **预期**：手牌空 + drawnCard 用掉 → 本回合直接结束，轮到下家，不死锁。

## 7. 我 review 过、判定为「可接受的设计取舍」的点（不算 bug，列出供你拍板）

1. **最后一轮自己回合食胡失败**：会触发 game-over，winner 由排名推断。此时该玩家手牌 +1（退回的摸牌）、归档没加 → 通常排名垫底。这是设计取舍（旧版允许最后补一组，新版一律不补）。**若你觉得太狠，可改。**
2. **牌库摸空兜底**：抽牌堆和弃牌堆同时空 → 强制 game-over 防死循环。极端发牌（全员手牌拉满）下可能提前结束，属已知边界，与 dummy 数量研究相关（见另一份报告）。
3. **加重罚停轮数偏重**：自摸碰失败实际罚约「2 整轮 claim + 1 跳过」，比普通碰失败更重。设计如此（信息更多惩罚更重）。**若测试中觉得劝退，可调。**
